Longitudinal symptom prediction

ANS-based multi-symptom prediction
from wearable physiology

An N-of-1 longitudinal study that trains five independent models — for severity, autonomic dysfunction, post-exertional malaise, fatigue, and brain fog — from nightly Polar wearable data and a structured clinical diary. Every model retrains automatically when a new diary entry is logged.

What this project actually is

A fully automated end-to-end prediction pipeline

This is not a chart or a one-time analysis. It is a working system that runs automatically, connects raw wearable data to clinical symptom outcomes, and retrains whenever new data arrives.

01 Nightly data ingest. A GitHub Actions workflow fetches raw RR-interval data from the Polar AccessLink API at 06:00 UTC every night. Artifact filtering applied (valid range: 300–2000 ms). 193 valid nights from 243 monitored.
02 Advanced HRV extraction. Raw RR intervals are processed by compute_hrv.py using neurokit2, producing 8 time-domain, frequency-domain, and nonlinear metrics per night: SDNN, pNN50, RMSSD, LF power, HF power, LF/HF ratio, SD1, SD2, DFA-alpha1. These metrics are not available from Polar’s proprietary pipeline.
03 Daily dataset linkage. merge_hrv.py links computed HRV features to the symptom diary (diary_live.csv), building lag windows at t0, t−1, and t−2 for each of 13 candidate features (39 combinations per target). Each new diary push triggers a full dataset rebuild.
04 Multi-target model training. retrain_predictor.py runs forward greedy feature selection independently for each of 5 symptom targets. Validation is LOO-CV with 1000× bootstrap CI. Three algorithms evaluated per target (LogisticRegression, RandomForest, GradientBoosting) — best selected by LOO-CV AUC.
05 Results committed to polar_live.json. Predictor results, feature weights, per-target AUCs, and CI bounds are auto-committed to the repository via GitHub Actions. This JSON is the single source of truth for all live data on this site.
06 Live web display. The metrics shown at kineticaai.com/ans-predictor.html are read directly from polar_live.json. No manual updates, no stale screenshots.

Study design

DesignProspective longitudinal N-of-1
SubjectSingle adult, post-Lyme / autonomic dysfunction
Monitoring period243 days continuous (Polar Grit X2, nocturnal PPG)
Diary entries61 (DIARY_v2 — prospective, structured, 7-domain)
Effective pairs61 (55 for autonomic dysfunction — missing HRV nights)
Symptom domains7 recorded: severity, PEM, fatigue, brain fog, autonomic, pain, mood
Prediction targets5 modelled: severity, PEM, fatigue, brain fog, autonomic dysfunction
WearablePolar Grit X2 · GDPR JSON export (ppi_samples, RR intervals in ms)
HRV computationneurokit2: SDNN, pNN50, RMSSD, LF/HF, HF power, SD1, SD2, DFA-alpha1
Candidate features13 features × 3 lag windows = 39 combinations per target
Feature selectionForward greedy per target · max 5 features · stop if AUC gain < 0.01
Primary validationLeave-One-Out CV (LOO-CV) — N iterations, no data leakage from scaling
UncertaintyBootstrap 1000× for 95% CI on AUC per target
AlgorithmsLogisticRegression (L2), RandomForest (100 trees), GradientBoosting (100 est.)
Target binarizationSeverity ≥6, Autonomic ≥5, PEM ≥5, Fatigue ≥6, Brain fog ≥5 (0–10 scale)

Missing data is MNAR, not imputed. The worst symptom days have no diary entries because autonomic fatigue prevented recording. This is Missing Not At Random — the missingness mechanism is the phenomenon. No retroactive entries were fabricated. All 61 pairs are prospective, without confirmation bias.

Bootstrap over SMOTE. Residual correlations (rho = +0.547 for brain fog, rho = +0.372 for autonomic dysfunction against severity residuals) demonstrate meaningful clinical structure. Synthetic oversampling would destroy it. Bootstrap resampling preserves the original data distribution while quantifying uncertainty.

Multi-target prediction performance

Five independent models, each discovering its own optimal feature set from 39 candidates. Logistic Regression was the best algorithm for all 5 targets. Results from the live pipeline; the reproducible version (seed=42, published data) is within bootstrap CI for all targets.

⚠ Brain fog class split: 54 positive / 6 negative. AUC 0.99 is likely inflated — the model may predict the minority class by exclusion rather than signal. Reported for transparency, not as a headline result.

Key finding: physiological coherence of the autonomic dysfunction model. Autonomic dysfunction is the only target that selected advanced neurokit2-derived features in the live pipeline: LF/HF ratio (sympathovagal balance, lag−1) and SD1 (short-term Poincaré variability, lag−0). These features are not available from Polar’s proprietary metrics. A model that predicts autonomic dysfunction using direct measures of autonomic balance is not just statistically significant — it is physiologically coherent. Deployment headline: AUC 0.84 [0.72, 0.94], N=55.

Residual analysis: the model’s error is clinical signal. Severity model residuals (prediction error) correlate with symptom dimensions the nocturnal HRV features cannot capture: brain fog (ρ = +0.547, p < 0.001) and autonomic dysfunction (ρ = +0.372, p = 0.006). The failures of the model point toward the neurological dimensions that ANS features alone cannot explain.

Reproducibility. Running scripts/retrain_predictor.py from the published data (seed=42) yields AUC 0.837 for autonomic dysfunction. The live pipeline at kineticaai.com (0.829) uses a cleaner L4 feature extraction path with tighter data provenance. Both results are within the bootstrap CI [0.72, 0.94]. All code is public.

Evolution of the study

From early analysis to multi-target system

▸ V1 — ARCHIVED · Early analysis (Mar 2026)

Severity predictor from 34 observation pairs

The first model targeted a single outcome: high-severity days (≥7/10) predicted 48 hours in advance. Data came from two non-consecutive periods (Sep–Oct 2025 + Feb 2026), producing 34 valid HRV–symptom pairs using four features: ANS status, RMSSD, recovery indicator, and Zolpidem flag.

LOO-CV AUC-ROC: 0.656 · Sensitivity: 81.2% · Specificity: 66.7%.
Dominant feature: ans_status at lag−2 (ρ = +0.431, p = 0.011, n=34). Zolpidem was present in 24 of 34 entries — severe confounding, not corrected in this version.

The abstract (abstract_v1.md in the repository) was never submitted. These numbers are real and preserved for historical transparency. They should not be cited as the current system state. Design was retrospective; no hypothesis was pre-registered.

▸ V3 — CURRENT SYSTEM (continuous, auto-updated)

Multi-target system with forward selection per symptom domain

Model v3 extends the study in every dimension: 61 prospective diary entries across 243 days, 5 independent prediction targets, 13 candidate features (including 8 neurokit2-derived metrics not available from Polar’s proprietary pipeline), and forward greedy selection from 39 feature-lag combinations per target.

Key advances over v1: each symptom selects its own optimal predictors (no fixed feature set imposed); neurokit2 features enable the autonomic dysfunction coherence finding (LF/HF + SD1); bootstrap CI replaces point AUC; the full pipeline is automated and retrains on every diary push. Severity AUC rose from 0.656 (v1, 34 pairs, fixed features) to 0.92 (v3, 61 pairs, forward selection) with 5 features vs 4.

Live deployment

From diary push to live display

The metrics shown on this site are generated by the pipeline and auto-committed to polar_live.json — the single source of truth that feeds all live displays.

Polar Grit X2 → extract_rr.py → compute_hrv.py (neurokit2) → merge_hrv.py → retrain_predictor.py → polar_live.json → kineticaai.com

Two GitHub Actions workflows orchestrate this automatically:

polar-biometrics.yml — runs nightly at 06:00 UTC. Fetches fresh Polar AccessLink data, computes HRV features from raw RR intervals, and updates the nightly time series in polar_live.json.

polar-retrain.yml — triggers on every diary push. Rebuilds the full HRV–symptom dataset, runs forward selection and LOO-CV for all 5 targets, computes bootstrap CIs, and writes updated predictor results to polar_live.json via auto-commit.

The live display of AUCs, feature weights, and sparklines at kineticaai.com/ans-predictor.html reads directly from this file. The numbers you see there are connected to polar_live.json — not hardcoded.

Limitations (honest)
  • !N=1. All results are idiographic. No generalization beyond this individual.
  • !N=61 pairs. Preliminary. Bootstrap CIs are wide (e.g. severity [0.84, 0.99]).
  • !Brain fog AUC 0.99. Class split 54/6. Likely inflated — the model may learn to predict the minority class by exclusion, not signal.
  • !Consumer wearable PPG, not clinical ECG. RR interval accuracy depends on PPG quality and motion artifacts.
  • !MNAR assumption. Missing days are assumed worst — not independently verified.
  • !Fixed confounders, uncontrolled. Alprazolam 3 mg, Bupropion 300 mg, Pregabalin 500 mg throughout the study.
  • !Multiple comparisons. 5 targets tested independently without family-wise error correction.
  • !Correlation ≠ causation in all analyses.
